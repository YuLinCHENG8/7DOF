# ik_cycling_test.cpp 改动总结

## 1. theta_1 / theta_5 输出归一化

原来在肩部和腕部解算后，关节角经过 offset 转换后，打印值可能落到 `[-180°, 180°]` 之外。

现在在输出前补了归一化逻辑：

- `q_out[0]`、`q_out[2]`
- `q_out[4]`、`q_out[6]`

都会被强制 wrap 到 `[-180°, 180°]`。

这样可以避免出现类似 `356°` 这种看起来越界的结果。

## 2. 臂角区间搜索改为“只在当前区间内搜索”

原来的黄金分割搜索会在所有可行臂角区间上搜索，容易让解跳到别的区间。

现在改成：

- 先根据当前臂角 `psi_cur` 找它所在的区间
- 如果当前臂角不在任何区间里，就选最近的区间
- 只在这个区间内执行 `golden_section_1d()`

这样可以保证 IK 解更连续，不会跨区间跳变。

## 3. 当可行区间为空时，增加降级备选区间

当 `INTERVALS.size() == 0` 时，不再直接空跑，而是使用硬编码备选区间：

- `[0.0, 0.9021] rad`
- `[1.1019, 2.646297] rad`

这样可以避免区间为空导致的搜索失败。

## 4. 腕中心向量统一按 145 mm 处理

原先腕中心相关的向量写法是：

```cpp
sqrt(145^2 + 2.5^2)
```

后来确认这个写法会把 `2.5 mm` 的偏移重复算进腕中心长度里。

当前修正为只使用：

```cpp
(0, 0, 145)
```

也就是只保留末端到腕中心的 z 方向长度，`-2.5 mm` 的偏移已经包含在 DH / FK 的变换里，不需要在这一步重复加。

## 5. 结果

修正后：

- 关节角输出范围正常
- 区间搜索更稳定
- 腕中心计算与 FK 一致
- IK 结果的连续性更好

## 备注

如果后续还要继续排查 2.5 mm 相关误差，建议重点看：

- FK 的 MDH 变换是否和 IK 中的几何假设完全一致
- `Frame 6 -> Frame 7` 的坐标定义是否有额外偏置
- `compareTransforms()` 中打印的误差单位是否和实际一致
